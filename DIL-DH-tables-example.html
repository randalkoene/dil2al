DIL Dependency Hierarchy in HTML Tables

The following example shows how a DIL Dependency Hierarchy can be displayed in tables. There are some minor issues to be aware of. When a DIL entry is a dependency of multiple superiors and the drawn hierarchy is wide and deep enough to encounter it more than once, it will be drawn more than once. The number of occurrances can be indicated in the cell (pre-calculated during preparation of the drawing). While colours or the thickness of borders could be used to indicate the weight of a dependency, this will not show up in w3m, so that the weight should be included as a numerical parameter. There may be a limit to the number of tables within tables that browsers can display. The width of a table should be set in pixels so that cells do not become too small.

SUPERIOR
DEPENDENCY 1-1
DEPENDENCY 1-1-1
DEPENDENCY 1-2
DEPENDENCY 1-2-1
DEPENDENCY 1-2-1-1 DEPENDENCY 1-2-1-2
DEPENDENCY 1-2-1-3
DEPENDENCY 1-2-1-3-1
DEPENDENCY 1-2-2
DEPENDENCY 1-3
DEPENDENCY 1-3-1
DEPENDENCY 1-3-1-1 DEPENDENCY 1-3-1-2
